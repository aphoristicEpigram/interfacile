# New ticket

Turn a request into a complete ticket in the right epic. The `interfacile` CLI
owns ids, filenames, and frontmatter — never write those by hand.

## Steps

1. **Title + epic.** Both are required. If the epic is unclear, run
   `interfacile tickets` and ask which epic fits (or `interfacile epics` to
   create a new one).
2. **Intake.** Ask, briefly and in one message: context (why), approach (how,
   roughly), acceptance criteria (how we know it's done), dependencies
   (existing ticket ids, or none), and effort / risk / priority only if they
   deviate from the defaults (2h / LOW / 3).
3. **Draft.** Write the body — Context, Approach, Acceptance criteria as
   `- [ ]` checkboxes — grounded only in the answers given. Show the draft and
   stop for approval.
4. **Create.**

       interfacile new "<Title>" --epic E00N [--risk ... --priority N --effort ... --depends-on ID,ID]

   Then replace the template body of the created file with the approved draft,
   leaving the frontmatter untouched.
5. **Verify.** `interfacile lint` must pass. Report the ticket id and path.

## Rules

- Don't invent requirements the user didn't state; mark open questions as
  `**Open question:** ...` in the Approach section instead of guessing.
- Don't start implementing — this skill only creates the ticket
  (`work-ticket` does the work).
