# IF-0042 · Version-stamped skills with staleness nudge in lint

## Context

`interfacile skills` refreshes installed skills, but nothing told a repo it was
running stale ones — the same silent drift that let imported debt linger.

## Approach

Every generated file carries `<!-- generated by interfacile vX.Y.Z -->`.
`lint` compares installed markers against the running package and prints a
one-line nudge (also in `lint --json`); silent when a repo has no skills —
that's a choice, not a fault.

## Acceptance criteria

- [x] Generated skills and process doc carry the version marker.
- [x] `lint` nudges on version mismatch and on unversioned installs; quiet
      when current or absent (unit-tested).
- [x] `interfacile skills` clears the nudge.
